# คู่มือการใช้งานระบบ Attack Surface Management (ASM)

เอกสารฉบับนี้จัดทำขึ้นเพื่อเป็นคู่มือสำหรับผู้ใช้งานระบบ Attack Surface Management (ASM) โดยอธิบายขั้นตอนและวิธีการใช้งานฟังก์ชันต่างๆ ของระบบอย่างละเอียดตามลำดับขั้นตอน เพื่อให้ผู้ใช้งานสามารถบริหารจัดการและประเมินความเสี่ยงของสินทรัพย์ดิจิทัลภายนอกองค์กรได้อย่างมีประสิทธิภาพ

---

## 1. การเข้าสู่ระบบ (Login)

หน้าต่างสำหรับการผ่านกระบวนการยืนยันตัวตน เพื่อเข้าถึงข้อมูลและเครื่องมือภายในระบบ
**ฟีเจอร์เด่นประจำหน้า:**
*   **Authentication & Access Control:** ระบบตรวจสอบสิทธิ์ความปลอดภัย ป้องกันเซสชัน (Session) ด้วยกระบวนการ JWT (JSON Web Token) ควบคุมเฉพาะผู้ที่มีสิทธิ์เท่านั้น

**ขั้นตอนการใช้งาน:**
1. เข้าสู่หน้าจอ Login ผ่านเว็บเบราว์เซอร์
2. กรอกข้อมูลสำหรับเข้าสู่ระบบ โดยใช้บัญชีผู้ดูแลระบบเบื้องต้นดังนี้:
   * **ชื่อผู้ใช้งาน (Username):** `admin`
   * **รหัสผ่าน (Password):** `admin123`
3. คลิกปุ่ม **Login** เพื่อเข้าสู่ระบบ หากชื่อผู้ใช้งานและรหัสผ่านถูกต้อง ระบบจะนำท่านเข้าสู่พื้นที่การทำงานหลัก

---

## 2. หน้าจัดการโดเมนเป้าหมาย (Domain Discovery)

ระบบนี้ใช้สำหรับการค้นหาซับโดเมน (Subdomains) ของเป้าหมาย เพื่อขยายขอบเขตในการมองเห็นพื้นที่การโจมตี (Attack Surface)
**ฟีเจอร์เด่นประจำหน้า:**
*   **Automated Subdomain Enumeration:** ฟีเจอร์วิเคราะห์และดึงข้อมูลซับโดเมนทั้งหมดที่เกี่ยวข้องกับโดเมนหลักอัตโนมัติ (ผ่าน WhoisXML API)
*   **Asset Inventory Logging:** ระบบรวบรวมและจัดเก็บข้อมูลสินทรัพย์ใหม่ลงในคลังฐานข้อมูลกลางขององค์กร

**ขั้นตอนการใช้งาน:**
1. นำทางไปยังเมนู **Domain Discovery** 
2. ในช่องระบุโดเมนเป้าหมาย ให้กรอกชื่อโดเมนหลักขององค์กรที่ต้องการตรวจสอบ (ตัวอย่างเช่น `example.com`)
3. กดปุ่มเพื่อเริ่มทำการค้นหา (Discover) ระบบจะทำการค้นหาโดเมนย่อยทั้งหมดที่เกี่ยวข้องกับระดับองค์กรนั้น
4. เมื่อแอปพลิเคชันค้นหาและแสดงโดเมนเนมขององค์กรทั้งหมดขึ้นมาบนหน้าจอเรียบร้อยแล้ว ให้คลิกปุ่ม **Save to DB** เพื่อทำการจัดเก็บข้อมูลสินทรัพย์ทั้งหมดลงในฐานข้อมูลของระบบสำหรับการนำไปประมวลผลในขั้นตอนต่อไป

---

## 3. หน้าติดตามตำแหน่งทางภูมิศาสตร์ของสินทรัพย์ (Digital Footprint)

หน้านี้จะดูแลข้อมูลที่เกี่ยวกับร่องรอยทางดิจิทัลและข้อมูลระบุตำแหน่งทางภูมิศาสตร์ (Geolocation) ของสินทรัพย์
**ฟีเจอร์เด่นประจำหน้า:**
*   **Geolocation Tracking:** ดึงข้อมูลที่อยู่ฝั่งเซิร์ฟเวอร์ และพิกัดทางภูมิศาสตร์ ผ่านการทำงานร่วมกับ Shodan API 
*   **Interactive Map Visualization:** การพล็อตจุดพิกัดของสินทรัพย์แบบ Interactive บนกราฟิกแผนที่โลกเสมือนจริง

**ขั้นตอนการใช้งาน:**
1. นำทางไปยังเมนู **Digital Footprint**
2. คลิกที่ปุ่ม **Sync Data** เพื่อสั่งให้ระบบทำการค้นหาข้อมูลที่ตั้ง (Location) และรายละเอียดเพิ่มเติมของสินทรัพย์แต่ละรายการที่ได้ลงทะเบียนไว้ในฐานข้อมูล (จากการ Save to DB ในขั้นตอนที่ 2)
3. **การแสดงผล:** 
   * **แผนที่ (Map):** ข้อมูลตำแหน่งที่ตั้งของเซิร์ฟเวอร์หรือสินทรัพย์จะถูกจุดและแสดงผลบนแผนที่โลกทางภูมิศาสตร์ เพื่อให้เห็นภาพการกระจายตัวของสินทรัพย์
   * **ตารางข้อมูล (Table):** แสดงรายละเอียดแบบเจาะจงของแต่ละ IP/Domain ที่พบบนระบบ 

---

## 4. หน้าจัดการและตรวจสอบช่องโหว่ (Issues Profile)

ทำหน้าที่เป็นศูนย์กลางในการสแกน รายงาน และดูแลความเสี่ยงหรือช่องโหว่ที่เกิดขึ้นกับระบบขององค์กร
**ฟีเจอร์เด่นประจำหน้า:**
*   **Python Engine Integration:** การทำเธรด (Thread) สั่งการโมดูลรันสแกนเนอร์ภาษาไพธอนจากฝั่ง UI ไปยังเซิร์ฟเวอร์
*   **Automated Issue Categorization & Tracking:** การจัดหมวดหมู่ช่องโหว่อัตโนมัติและระบบติดตามสถานะ (Status) การดำเนินการแก้ไขช่องโหว่

**ขั้นตอนการใช้งาน:**
1. นำทางไปยังเมนู **Issues Profile**
2. คลิกปุ่ม **Scan All Assets** เพื่อเริ่มกระบวนการสแกนหาช่องโหว่ โดยระบบจะทำการเรียกใช้งานโมดูลไพธอน (Python Modules) ที่ออกแบบไว้ เพื่อตรวจสอบสินทรัพย์ที่มีอยู่ในระบบทีละรายการ
3. **วิธีการดูตารางและการแสดงผล:** 
   ข้อมูลที่สแกนพบจะถูกนำมาแสดงลงในตาราง โดยมีคอลัมน์ที่สำคัญดังนี้:
   * **Target/Asset:** สินทรัพย์ (IP หรือ Domain) ที่กำลังถูกอ้างอิง
   * **Issue/Vulnerability:** ชื่อหัวข้อหรือประเภทของช่องโหว่ที่ตรวจพบ
   * **Severity:** ระดับความรุนแรงของช่องโหว่ (เช่น High, Medium, Low)
   * **Date Discovered:** วันและเวลาที่ระบบค้นพบ
4. **การตรวจสอบเชิงลึกและการปฏิบัติการ:** 
   ผู้ใช้สามารถคลิกเข้าไปดูรายละเอียด (View Details) ของช่องโหว่แต่ละรายการได้ เพื่อตรวจสอบข้อมูลด้านเทคนิคเพิ่มเติม และสามารถจัดการสถานะของช่องโหว่ในหน้านี้ได้ (เช่น การทำเครื่องหมายว่ารับทราบข้อบกพร่องแล้ว หรือ การเปลี่ยนสถานะความคืบหน้าในการแก้ไข)

## 5. หน้าปัจจัยและเกณฑ์คะแนนความเสี่ยง (Score Factor)

ฟังก์ชันนี้จัดทำเพื่อจัดระดับความเสี่ยง โดยการให้คะแนนและจัดหมวดหมู่ให้เหมาะสมกับการวัดผล
**ฟีเจอร์เด่นประจำหน้า:**
*   **Dynamic Risk Weight Configuration:** อนุญาตให้ผู้ดูแลระบบให้น้ำหนักความสำคัญ (Weight) และระบุค่าคะแนนกับโมดูลใหม่ๆ เพื่อให้ระบบนำไปคำนวณแบบพลวัต (Dynamic) 

**ขั้นตอนการใช้งาน:**
1. นำทางไปยังเมนู **Score Factor**
2. ภายในหน้าจอนี้ ผู้ใช้งานสามารถจัดหมวดหมู่และปรับแต่งค่าน้ำหนักความสำคัญ (Weight) ของโมดูลการสแกนความเสี่ยงต่างๆ ที่ระบบพัฒนาไว้ ให้เข้ากับบริบทและความเข้มงวดขององค์กร
3. **การใช้งานตาราง:** 
   ตารางจะแบ่งหมวดหมู่ปัจจัย (Factors) ตามชนิดของช่องโหว่ ซึ่งระบบสรุปคะแนนผลกระทบออกมา ผู้บริหารหรือผู้ใช้งานที่มีสิทธิ์ สามารถแก้ไขและปรับเปลี่ยนคะแนนหรือการให้คะแนนช่องโหว่แต่ละระดับได้ผ่านหน้าจอนี้ เพื่อให้การคำนวณเกรดรวมของระบบมีความแม่นยำที่สุด

---

## 6. หน้าสรุปผลการปฏิบัติการ (Dashboard)

ทำหน้าที่เป็นหน้าจอหลัก (Home) หลังจากการลงชื่อเข้าใช้งาน มีองค์ประกอบสำหรับการติดตามภาพรวมของระบบ
**ฟีเจอร์เด่นประจำหน้า:**
*   **Security Grading System:** การนำค่าคะแนนความเสี่ยงและปริมาณช่องโหว่ทั้งหมดที่ได้จากระบบ มาแปลงค่าเป็น "เกรดความมั่นคงปลอดภัยองค์กร" เพื่อง่ายต่อการรับรู้
*   **Security Analytics Charts:** ระบบวาดกราฟและสรุปยอดสถิติให้โดยอัตโนมัติ

**ขั้นตอนการใช้งาน:**
1. นำทางไปยังเมนู **Dashboard**
2. **องค์ประกอบ (Components) และความหมาย:**
   * **Security Grade (ระดับคะแนนความปลอดภัย):** แสดงเกรดประเมินผลรวมของสภาพแวดล้อมดิจิทัลขององค์กร โดยคำนวณจากเปอร์เซ็นต์ปัญหาที่พบเทียบกับ Score Factor
   * **Total Assets (สินทรัพย์ทั้งหมด):** แสดงจำนวนทรัพยากร ทั้งโดเมน และเซิร์ฟเวอร์ที่อยู่ภายใต้การสอดส่องของระบบ
   * **Active Issues / Vulnerability Count (จำนวนความเสี่ยง):** สรุปยอดรวมของช่องโหว่ที่กำลังเป็นประเด็นอยู่ ณ ปัจจุบัน
   * **Charts / Graph (กราฟแสดงผล):** แผนภูมิสรุปรูปแบบของจุดอ่อนที่พบบ่อยระดับองค์กร เพื่อการตัดสินใจและจัดสรรทรัพยากรแก้ไขได้อย่างทันท่วงที

## 7. หน้าจัดการผู้ใช้งาน (User Management)

หน้าสำหรับควบคุมและจัดการสิทธิ์ของผู้ที่สามารถเข้าถึงระบบ 
**ฟีเจอร์เด่นประจำหน้า:**
*   **Role-based Access Control (RBAC):** ระบบจัดการบทบาท ควบคุมสิทธิ์การเข้าถึงข้อมูลว่าพนักงานระดับใดจะเห็นได้แค่หน้าจอใด
*   **Account Suspension:** ความสามารถในการยับยั้งบัญชีฉุกเฉิน กรณีพนักงานลาออกหรือล็อกอินต้องสงสัย

**ขั้นตอนการใช้งาน:**
1. นำทางไปยังเมนู **User Management**
2. ผู้ดูแลระบบ (Administrator) สามารถเห็นรายชื่อพนักงานหรือผู้ใช้งานที่ลงทะเบียนอยู่ในระบบได้
3. สามารถดำเนินกระบวนการ **เพิ่มผู้ใช้งานใหม่ (Add User)**, **แก้ไขสิทธิ์การใช้งาน (Edit Roles)** รวมไปถึง **ระงับการเข้าถึง (Deactivate/Delete)** ของบัญชีที่ไม่ได้รับอนุญาต เพื่อรักษาความปลอดภัยภายในของระบบ (Internal Security) 

---

## 8. หน้าการตั้งค่า (Settings)

หน้าตั้งค่าระบบโดยรวมสำหรับการปรับแต่งแอปพลิเคชัน 
**ฟีเจอร์เด่นประจำหน้า:**
*   **Third-party API Key Management:** การจัดการรหัสผ่านและคีย์ API นอกระบบ (เช่น Shodan) สำหรับแก้ไข/อัปเดต โดยไม่ต้องเข้าไปแก้ Source Code
*   **UI Theme Toggle:** การจดจำตัวเลือกอินเทอร์เฟซผู้ใช้แบบความสว่าง (Light Mode) หรือมืด (Dark Mode) ตามความถนัดแบบบุคคล

**ขั้นตอนการใช้งาน:**
1. นำทางไปยังเมนู **Settings**
2. ภายในหน้านี้ผู้ใช้สามารถตั้งค่าตัวแปรระดับระบบหรือค่าแวดล้อม (Environment parameters) ได้ 
3. ข้อมูลที่สามารถจัดการได้ รวมถึงการจัดการคีย์เชื่อมต่อของ API ฝ่ายที่สาม การบันทึกรูปแบบของ UI ธีมสี (Light/Dark Theme) และปรับแต่งพฤติกรรมพื้นฐานของการทำงานภายในระบบ
